# 第二阶段高价值收束训练设计文档

- 日期：2026-04-19
- 阶段：Brainstorming / 设计定稿
- 适用范围：Mini Challenge 第二阶段 `Closure / elementary_closure_v1`
- 相关技能约束：本设计先经过 `brainstorming` 梳理，基于现有 `Advanced + Closure` 双阶段系统；本文件不包含实现代码

## 1. 设计目标

第二阶段不是“小学计算第二遍”，也不是“更难一点的杂烩卷”。

它要承担的是：

1. 在不丢第一阶段成果的前提下，继续收束小学阶段所有与计算有关的重点难点。
2. 用尽量短的时间，把最有价值、最影响后续中学学习的能力优先打透。
3. 让孩子形成稳定的数感、结构感、方法选择能力和结果判断能力。
4. 让计算真正成为优势项目，而不是停留在“会做一些题”。

这个阶段的最终目标是：

`把小学计算训练收束到“稳、快、会判断、会迁移”，并顺利接到初中前的学习需求。`

## 2. 设计前提

本设计必须和现有系统一起分析，而不是孤立地看第二阶段。

### 2.1 第一阶段和第二阶段的关系

当前系统已经明确分成：

- `Advanced / 第一阶段`
- `Closure / 第二阶段`

其中：

- 第一阶段负责把核心算法练稳
- 第二阶段不再重复低效刷题
- 第二阶段要在第一阶段成果之上，承担：
  - 少量单点保温
  - 大量综合重组
  - 有针对性的重点强化

因此，第二阶段对第一阶段已练内容的原则是：

`双线并行`

也就是：

1. 一条线做少量保温，防止底盘掉线；
2. 一条线做综合重组，检验第一阶段能力是否已经能迁移。

### 2.2 不采用平均主义出题

第二阶段不能按“知识点平均铺开”的思路设计。

它必须显式区分：

- `高价值概念`
- `高价值题型`
- `低回报重复`

否则就会出现：

`丢了西瓜捡芝麻`

## 3. 第二阶段的两条主轴

第二阶段建议围绕两条并列主轴展开。

### 3.1 枢纽表征主轴

这一条主轴包括：

- 小数
- 分数
- 百分数
- 比
- 互化
- 比较

这是从六年级往初中看最关键的一组枢纽内容，因为很多后续的比率、比例、代数感、函数感都依赖这一组量感和表示转换是否真正打通。

### 3.2 复杂运算主轴

这一条主轴包括：

- 四则混合运算
- 小数四则混合
- 分数四则混合
- 小数分数混合
- 乘除优先
- 运算律
- 凑整、拆分、约分提速
- 后期引入中括号、大括号

这一条主轴不能漏掉，而且属于重点，因为它最能检验：

- 数感
- 运算顺序感
- 结构识别
- 方法选择
- 稳定性

### 3.3 两条主轴的关系

第二阶段不是先练完第一条主轴再练第二条主轴，而是：

- 全程同时存在两条主轴
- 前期以 `表征打通 + 轻量复杂混合` 为主
- 中期以 `综合重组 + 方法选择` 为主
- 后期以 `复杂混合 + 括号结构 + 稳定迁移` 为主

## 4. 三阶段训练弧线

第二阶段继续保留现有三段式弧线，但要明确每一段的训练任务。

### 4.1 收口期

目标：

- 把第一阶段成果接进第二阶段
- 把小数、分数、百分数、比真正接通
- 建立“看到题先看结构”的习惯

特点：

- `核心保温区` 比例略高
- `表征互通` 高频出现
- `复杂四则混合` 全程进入，但以轻量版出现
- `单位 / 率 / 关系综合` 开始进入，但先以基础整合为主
- `重点强化区` 以查漏补缺为主

一句话总结：

`先把桥搭起来。`

### 4.2 收束期

目标：

- 让不同表示和方法在混合题里真正切换起来
- 明显提高复杂混合运算、单位率关系综合的比重
- 把能力真正拧成一股

特点：

- `表征互通` 继续高频，但不只是会转，而是“转完还要判断、还要算”
- `复杂四则混合` 明显加重
- `单位 / 率 / 关系综合` 明显加重
- `核心保温区` 压缩到更小
- `重点强化区` 更偏高价值错点，而不是泛化补漏

一句话总结：

`开始真正收束能力。`

### 4.3 毕业判定期

目标：

- 检查孩子是否已经形成高手型计算能力
- 检查结构感、方法感、数感、稳定度是否同时在线

特点：

- `表征互通` 更偏边界判断和结果判断
- `复杂四则混合` 进入完整版本，引入中括号、大括号、多层结构
- `单位 / 率 / 关系综合` 更偏综合检验
- `核心保温区` 最小化
- `重点强化区` 变成最后的短板追打区

一句话总结：

`不是再教，而是检验有没有真正毕业。`

## 5. 第二阶段每天两页的固定骨架

第二阶段每天两页不应平均铺知识点，而应固定成 `4 + 1` 结构：

1. `Section I · 核心保温`
2. `Section II · 表征互通主训练`
3. `Section III · 复杂四则混合主训练`
4. `Section IV · 单位 / 率 / 关系综合`
5. `Section V · 重点强化`

这个骨架保持稳定，让孩子有熟悉感；而每个 section 内部的题型比重和难度随阶段变化。

### 5.1 Section I · 核心保温

角色：

- 保住第一阶段成果
- 做底盘检查
- 避免基础掉线

适合放：

- 小数乘除基本稳定题
- 分数基本运算保温题
- 少量整数 / 竖式稳定题
- 少量简算顺序题
- 少量“先判断大小再算”的短题

原则：

- 题量小
- 难度中低
- 只做保温，不做主体

### 5.2 Section II · 表征互通主训练

角色：

- 第二阶段第一主战场
- 打通小数 / 分数 / 百分数 / 比 / 互化 / 比较

适合放：

- 互化
- 比较大小
- 接近 `0 / 1/2 / 1`
- 互化后再比较
- 互化后再计算
- 同一个量换表示
- 判断哪种表示更方便算

原则：

- 每天都要有
- 前期更偏“打通”
- 后期更偏“判断 + 迁移”

### 5.3 Section III · 复杂四则混合主训练

角色：

- 第二阶段第二主战场
- 建立结构感、顺序感、方法感
- 先训练“该怎么做更快更稳”，再升级结构复杂度
- 让孩子从“硬算型”走向“会看结构、会选方法”的高手型计算

适合放：

- 小数混合四则
- 分数混合四则
- 小数分数混合
- 乘除优先
- 运算律参与
- 凑整拆分
- 约分提速
- 后期中括号、大括号
- “哪种做法更快 / 更稳”的方法判断题

原则：

- 全程都有
- 前期轻量
- 中期明显加重
- 后期进入结构化完整版本
- 方法选择优先于纯结构堆叠
- 结果判断必须嵌入这一块，不能只剩执行

#### Section III 的四个内部子题族

##### M1. 结构识别型

- 看出哪一部分适合先算
- 看出哪一组可以凑整
- 看出哪一段适合先约分
- 看出哪一道更适合先互化

作用：

- 直接提高模式识别
- 降低“拿到题就硬算”的倾向

##### M2. 方法选择型

- 先通分还是先约分
- 先互化还是直接算
- 先拆分还是先合并
- 先化成整数除法还是直接做小数除法

作用：

- 这是 Section III 最该优先打厚的一类
- 它决定孩子会不会真正变成高手型计算者

##### M3. 复杂执行型

- 小数分数混合四则
- 连乘连除
- 运算律参与
- 后期中括号、大括号
- 多层结构题

作用：

- 把前两类识别和选择真正落到执行里
- 但不能独大，否则会退回“难算题堆砌”

##### M4. 结果判断型

- 结果大概在哪个区间
- 是否可能大于 `1`
- 是否可能比原数更大
- 小数点位置是否合理

作用：

- 把数感真正拴进复杂混合运算
- 避免“算得很辛苦但最后还错而不知”

#### 三阶段里 M1-M4 的重心变化

`收口期`

- `M1` 高
- `M2` 高
- `M3` 中低
- `M4` 中

重点：

- 先让孩子学会“不乱算”

`收束期`

- `M1` 中
- `M2` 高
- `M3` 高
- `M4` 中高

重点：

- 方法选择和复杂执行一起拉起来

`毕业判定期`

- `M1` 中
- `M2` 高
- `M3` 高，进入完整括号结构
- `M4` 高

重点：

- 检查是不是已经形成“高手型复杂计算能力”

#### Section III 的稳定题型模板

1. 小数四则混合
2. 分数四则混合
3. 小数 + 分数混合
4. 运算律 / 凑整 / 拆分 / 约分提速
5. 括号结构升级
6. 方法辨别与结果判断嵌入题

其中最应长期高频打磨的 3 组是：

- `小数 + 分数混合`
- `运算律 / 凑整 / 拆分 / 约分提速`
- `括号结构升级`

这些题型最能同时训练：

- 数感
- 结构感
- 方法感
- 结果判断
- 向初中过渡所需的稳定计算优势

### 5.4 Section IV · 单位 / 率 / 关系综合

角色：

- 把纯计算接到更高位关系结构
- 为初中过渡做铺垫

适合放：

- 单位换算
- 单价 / 速度 / 平均量
- 百分率 / 折扣
- 比和比例关系
- 方程和数量关系结合
- 先统一单位再算
- 先判断关系再列式

原则：

- 不是天天最大头
- 但周期高频回看
- 中后期权重明显增加

### 5.5 Section V · 重点强化

角色：

- 用最少题量打最值钱的薄弱点
- 是效率最高的一块

适合放：

- 当前最强 gap 对应题
- 最近错题变式
- 第一阶段遗留薄弱点
- 第二阶段 A 类核心短板
- 边界题
- 反向题
- 方法辨别题
- 结果判断题

原则：

- 不固定知识点
- 每天个性化
- 题量少，但价值最高

## 6. A / B / C 三层价值划分

第二阶段题型必须显式分成 A / B / C 三层。

### 6.1 A类：必须反复打磨

#### A1. 表征互通核心

- 小数、分数、百分数、比互化
- 比较大小
- 接近 `0 / 1/2 / 1` 的量感判断
- 互化后再比较、再判断、再计算

原因：

- 这是小学到初中的总枢纽
- 后面很多比例、函数感、代数感都要靠这一层打通

#### A2. 复杂四则混合

- 小数四则混合
- 分数四则混合
- 小数分数混合
- 乘除优先
- 运算顺序
- 运算律
- 凑整、拆分、约分提速
- 后期中括号、大括号
- 方法选择优先于纯执行堆叠
- 结果判断嵌入题

原因：

- 这是结构识别和方法选择的主要载体
- 最能拉开“会算”和“高手型计算”的差距

#### A3. 数感与结果判断

- 商比 `1` 大还是小
- 结果数量级判断
- 小数点位置判断
- 结果是否合理
- 先估再算、先判再算

原因：

- 这是初中前极关键的底层能力
- 没有这层能力，后面很容易机械算、低效算、错而不知

#### A4. 方法选择题

- 该通分还是先约分
- 该互化还是直接比
- 该硬算还是先凑整
- 该先统一单位还是先列式

原因：

- 这是模式识别真正落地的地方
- 高手与普通训练者的差距往往就在这里

### 6.2 B类：周期回看

#### B1. 单位 / 率 / 速度 / 平均量

- 单位换算
- 单价、速度、平均量
- 百分率、折扣
- 比和比例关系

原因：

- 很重要
- 但更适合承担“综合接桥”和“应用结构”的角色
- 不必像 A 类那样天天主打

#### B2. 方程与数量关系

- 逆运算思路
- 未知数位置
- 数量关系转方程
- 方程和比率、百分率、单位题结合

原因：

- 是很重要的过渡能力
- 但在第二阶段里更适合做综合检验，不必占主战场过多题量

#### B3. 竖式稳定保温

- 竖式乘除
- 整数四则稳定性
- 少量基础简算保温

原因：

- 第一阶段已经练了很多
- 第二阶段只需要确认没掉线

### 6.3 C类：够用即可

#### C1. 纯单线重复基础题

- 只是换数字的同类基础题
- 不需要判断和选择的直接计算题

#### C2. 太碎的边角知识点

- 单独存在感很强
- 但不值得占每日主体题量的知识点

#### C3. 只考记忆、不考结构的题

- 只背表面规则
- 不涉及量感、关系、方法选择的内容

原则：

- 不是完全不要
- 但只做最低必要覆盖
- 不允许挤占 A 类时间

## 7. 选题原则

### 7.1 要大量出的题

#### 1. 能同时检验两个以上核心能力的题

例如：

- 互化后再比较
- 互化后再计算
- 单位统一后再列式
- 分数小数混合四则
- 结果先判断再精算

原因：

- 一道题能同时练数感、结构感、方法选择和稳定性
- 训练效率高

#### 2. 能暴露方法选择的题

例如：

- 先通分还是先约分
- 先互化还是直接比
- 硬算还是先凑整
- 先统一单位还是先列式

原因：

- 这类题最能区分“会做题”和“会想题”

#### 3. 能暴露数感和结果感的题

例如：

- 商大于 `1` 还是小于 `1`
- 结果更接近 `0 / 1/2 / 1`
- 小数点大概落在哪
- 结果是否合理

原因：

- 这是后续学习极高价值的底层能力

#### 4. 能把第一阶段成果重新编织起来的题

例如：

- 竖式能力进入复杂混合
- 小数计算进入单位和率
- 分数运算进入互化与比较
- 基础简算进入复杂结构题

原因：

- 第二阶段不是重新练第一阶段，而是把第一阶段成果变成可迁移能力

### 7.2 要少出的题

#### 1. 单一知识点、单一路径、变化很小的题

处理原则：

- 可以进入 `Section I` 做保温
- 不能占主体

#### 2. 只有情境壳变化、内核没有升级的题

处理原则：

- 少出
- 不被“看起来综合”误导

#### 3. 复杂但低价值的题

处理原则：

- 少出
- 我们要的是高价值难度，不是无效难度

### 7.3 要淘汰或极低频的题

#### 1. 纯机械重复题

- 一看就知道怎么做
- 只是重复熟练

#### 2. 只考死记硬背、不考理解的题

- 只背表面规则
- 不涉及量感、结构感、方法感

#### 3. 对后续价值不高但很占题量的边角题

原则：

- 只保留最低必要覆盖
- 不允许挤占 A 类题空间

## 8. 第二阶段的核心筛题标准

以后第二阶段题目是否值得进入主线，建议都用下面五条筛一次：

1. 这题能不能帮助建立 `数感`
2. 这题能不能帮助识别 `模式和结构`
3. 这题能不能逼孩子做 `方法选择`
4. 这题能不能把第一阶段能力 `迁移出来`
5. 这题对六年级到初中的后续学习是不是 `高价值`

占得越多，越值得保留。

如果一道题只是“会算一下”，但不承担这五条中的核心价值，就不该占太多位置。

## 9. 对现有第二阶段实现的影响

当前 `Closure` 已经有：

- 三阶段弧线
- 五类 gap 信号
- `Section V` 重点强化区
- `L1 / L2 / L3` 回炉
- 共享 `qualityFamily`

本设计不是推翻这些结构，而是要求：

1. 重新校准 `Section II / III / IV / V` 内部题型权重；
2. 显式把“复杂四则混合运算（含小数分数混合、运算律、括号升级）”提升到 A 类主轴；
3. 在 `Section III` 内部明确分出 `M1-M4` 四个子题族，并把“方法选择优先”写成明确规则；
4. 把 `Section III` 的稳定题型模板固定成 6 组，不再允许随机题池把复杂混合区稀释成泛泛杂烩；
5. 让 `Section I` 更明确承担保温角色，不再漂移成主体训练；
6. 让 `Section IV` 明确承担“关系综合”和“初中过渡桥接”角色；
7. 让 `Section V` 优先服务 A 类高价值短板，而不是泛化补漏。

## 10. 成功标准

如果第二阶段按本设计优化完成，应能达到：

1. 孩子不会感觉在“重新刷第一阶段”，但第一阶段成果不掉线。
2. 每天两页主体时间明显集中在最值钱的能力上，而不是平均铺开。
3. 小数 / 分数 / 百分数 / 比 / 互化 / 比较 与复杂四则混合运算成为真正的第二阶段主战场。
4. 复杂混合运算从头到尾都在，但前期轻、中期强、后期结构化升级。
5. 第二阶段更明确地培养：
   - 数感
   - 模式识别
   - 方法选择
   - 综合迁移
   - 稳定正确
6. 训练结果更符合“六年级往初中看”的高价值计算路线，而不是小学知识点平均重复。

## 11. 非目标

1. 不把第二阶段扩成新的题海系统。
2. 不要求所有小学知识点都平均分配同等训练时间。
3. 不为了“看起来难”而加入低价值复杂题。
4. 不把复杂四则混合运算做成单独孤岛，而是必须和数感、互化、方法选择一起协同。
5. 不在本设计阶段直接规定精确题量，而是先确定题型价值层级和 section 骨架。

## 12. 结论

第二阶段应被正式定义为：

`一个高价值计算收束系统`

它的工作不是再做一遍小学计算，而是：

- 保住第一阶段成果
- 打透最值钱的枢纽能力
- 把复杂混合运算做成优势
- 让孩子在小学阶段就形成对初中有延续价值的计算基本功

下一步应基于本设计，继续拆成一份明确的实施计划，落到：

- 现有 section 权重重排
- A / B / C 题型矩阵接线
- 三阶段题型升级轨迹
- 与现有 `gap + replay + qualityFamily` 的接法
