# 本套报告错题变式跟训设计文档

- 日期：2026-04-19
- 阶段：Brainstorming / 设计定稿
- 适用范围：Mini Challenge 现有“本套报告 / Set Review”内新增一块“基于本套错题生成变式跟训”的内页能力
- 相关技能约束：本设计先经过 `brainstorming` 梳理，遵循现有单页系统结构与打印链路；本文件不包含实现代码

## 1. 背景与目标

当前系统已经具备完整的单套训练闭环：

- 生成本套 AB 训练卷
- 家长批改并提交
- 系统记录每位学生在本套里的错题、粗心题、原题、答案与复盘提醒
- 在 `本套报告 / Set Review` 中回看本套表现

但这里还缺一个非常高价值的“当下补透”动作：

- 不是只看错题列表
- 不是简单把原题再做一遍
- 而是基于本套已经错掉的题，立刻再生成一轮同错误点、同方法、但题面和答案都变化的新题，让孩子马上巩固

这个功能的目标不是“再出一整套卷”，而是利用本套已经暴露出来的错误，快速而高质量地进行一轮紧凑的跟训。

### 核心目标

1. 让家长在批改完本套之后，能够直接在 `本套报告` 里继续一轮针对性训练。
2. 让补充训练基于“本套真实错题”，而不是泛泛地再刷一套。
3. 让新题保持同知识点、同错误点、同方法，但不能是原题重做。
4. 让输出适合直接打印，作为训练内页使用。
5. 让答案参考单独放在后面，方便家长核对，也方便只打印题页。

### 非目标

1. 不把这个功能做成新的独立训练系统。
2. 不把它做成单独的新路由或新页面体系。
3. 不要求覆盖全量错题本，只聚焦“本套刚发生的错误”。
4. 不机械地一错一题复制追打。
5. 不做长篇讲解或复杂交互。

## 2. 现有系统事实与挂接点

本设计必须贴着当前系统结构实现，而不是另起炉灶。

### 现有入口

- 右上角菜单中已有 `📋 本套报告` 按钮
- 点击后调用 `window.showSetReview()`
- 内容展示在现有 `report-modal` 中

### 现有数据来源

当前提交链路已经完整记录了本套批改结果：

- `submitGrades(student)` 会收集本套答案页里的每道题
- 每条记录已经包含：
  - `tag`
  - `grade`
  - `info`
- `info` 中已经有：
  - 原题题干
  - 正确答案
  - 分区
  - 纸面题号
  - 复盘提醒

这些内容最终会在 `StorageDB.saveSession()` 中写入本套 `sessionRecord.details`。

### 现有报告结构

当前 `buildSetReviewSection(student, setNum, highlightStudent)` 已经能按学生渲染：

- 本套正确 / 粗心 / 概念错数量
- 本套错题表
- 原题、正确答案、复盘提醒

因此，新功能最合理的挂法是：

`直接作为本套报告内的新增板块`

而不是开一个全新的数据结构或页面入口。

## 3. 设计结论

本功能采用：

`本套报告内嵌式错题变式跟训`

具体表现为：

- 每个学生的 `本套报告` 下方，新增一个：
  - `本套错题变式跟训 / Variant Follow-up`
- 该板块基于本套错题自动生成一组短而精的变式题
- 同时在后面附一页家长可参考的答案页
- 提供单独打印入口

### 为什么选择这一方向

1. 路径最短：家长批改后无需离开 `本套报告`。
2. 反馈最及时：错误刚暴露出来，马上跟训，最符合高效巩固。
3. 对现有系统侵入最小：复用现有 modal、现有本套会话数据、现有打印链路。
4. 最符合用户目标：不是看报告就结束，而是看完立刻补这一轮错点。

## 4. 跟训页设计原则

这个板块的第一原则不是“多出题”，而是“高效补透”。

### 4.1 不是原题重做

不能把原题直接再拿给孩子做一遍。

必须满足：

- 同知识点
- 同错误点
- 同方法要求
- 题面变化
- 数字变化
- 答案变化

目标是判断：

`孩子是真的掌握了，还是只记住了原题。`

### 4.2 不机械按错题条数复制

不能简单按“错了几题，就再生成几题”。

更合理的方式是：

- 先识别本套错题里有哪些“错误点家族”
- 再按错误点合并后生成变式训练

例如：

- 同一套里错了 3 道都属于“小数除法里商的小数点定位”
- 不需要再机械出 3 道
- 应该归并成 2 道更有代表性的变式题

### 4.3 概念错优先，粗心错保温

训练优先级建议明确区分：

- `wrong / 概念错误`
  - 优先生成变式题
  - 题量可多一点
- `careless / 粗心失误`
  - 只少量补温
  - 更偏检查习惯和自检步骤

也就是说，这页首先服务“概念没掌握”，其次才是“容易滑”。

### 4.4 题量短、密、立即可练

这不是新的长卷子。

建议控制为：

- 每位学生约 `4 到 8` 题
- 错题少就少出
- 错题多时按错误点归并，不盲目扩张

它的角色是：

`本套训练后的短程跟训页`

而不是额外再出一套正式卷。

## 5. 页面结构

每位学生的 `本套报告` 新增区域建议固定为四层。

### 5.1 本套批改摘要

沿用现有内容：

- 正确 / 粗心 / 概念错数量
- 本套错题列表

这一层的作用是：

先知道“本套到底错了什么”。

### 5.2 跟训摘要条

错题表下新增一条紧凑摘要，说明本次变式训练的逻辑，例如：

- 本次跟训共 6 题
- 其中 4 题针对概念错
- 2 题针对粗心保温
- 主要覆盖：小数除法、分数运算、互化比较

这一层的作用是：

让家长一眼知道这一页“在补什么”，而不是直接掉进题目里。

### 5.3 变式训练题页

正式题目区建议按“紧凑内页”来设计：

- 不再像主训练卷那样分成完整 A/B 套
- 更像本套报告后的“跟训内页”
- 一页内优先装下同一位学生本轮跟训题

题目结构建议按错误点家族分小组，而不是按原卷顺序硬排。

例如：

- 小数除法跟训
- 分数运算跟训
- 互化与比较跟训

但视觉上要轻，不要做成复杂章节页。

### 5.4 参考答案页

答案统一放在后面，作为家长参考区。

答案页要求：

- 简洁
- 易对照
- 主要给答案，不强求详细过程
- 如有必要，可附一句极短的检查提醒

这样可以同时满足：

- 平时只打印题页给学生练
- 需要核对时再看答案页

## 6. 打印设计

这个功能本质上是训练内页，所以必须从一开始就按打印来设计。

### 6.1 打印入口

建议在本套报告的跟训板块顶部提供两个按钮：

- `打印变式训练`
- `打印变式训练 + 答案`

### 6.2 打印目标

打印输出应更像“老师即时补充的一页跟训单”，而不是网页截图。

要求：

- 题页可单独打印
- 题页和答案可组合打印
- 页面结构清楚
- 标题简洁
- 不把报告表格和跟训题混在同一打印页里

### 6.3 打印风格

风格应延续当前系统的考试卷 / 讲义风格：

- 干净
- 克制
- 不花哨
- 适合黑白打印
- 不依赖重背景色传递信息

## 7. 变式题质量标准

这个功能的成败，关键不在“能不能生成题”，而在题是不是打中了错点。

### 7.1 合格变式题的定义

一题合格的变式，至少要满足：

1. 针对同一个核心错误机制
2. 不允许孩子靠背原题过关
3. 保留同类判断过程
4. 允许换数字、换顺序、换呈现方式
5. 难度只轻微浮动，不能突然跨层

### 7.2 示例原则

如果原错题是：

- 小数除法里“小数点定位错”

则变式应该继续要求：

- 先判断商大小
- 再处理除数化整
- 再决定商的小数点位置

而不是换成一个完全不同的除法技能。

如果原错题是：

- 分数加减里“通分判断错”

则变式应继续打：

- 是否先通分
- 通分后再算
- 最后是否化简

而不是退化成特别简单的分数认识题。

## 8. 与现有训练系统的关系

这个板块不是新系统，而是当前系统训练闭环的一部分。

正确关系应该是：

- 主训练卷负责暴露问题
- 本套报告负责解释问题
- 本套错题变式跟训负责立刻补一轮
- 后续长期错题本和自适应训练再负责长期追踪

也就是说，这个板块的定位是：

`短程补透`

而不是：

- 永久错题本替代物
- 第二套完整训练卷
- 独立复习系统

## 9. 最小可交付版本

第一版建议尽量聚焦，不一次做重。

### 第一版包含

1. 在 `本套报告` 中新增“本套错题变式跟训”板块
2. 基于当前学生本套 `details` 生成变式题
3. 对 `wrong` 和 `careless` 做轻度区分
4. 生成题页
5. 生成参考答案页
6. 提供：
   - 只打印题页
   - 打印题页 + 答案

### 第一版不包含

1. 跨多套报告合并跟训
2. 复杂讲解卡片
3. 很重的个性化排程
4. 新的独立错题训练中心

## 10. 风险与控制

### 风险 1：题量膨胀，跟训页变成长卷

控制方式：

- 按错误点归并
- 控制每位学生题量在 4 到 8 题

### 风险 2：变式题只是换数字，训练价值不够

控制方式：

- 强调“同错误机制”，不是“同表面形式”
- 优先走已有题族 / tag / family 的方法一致变式

### 风险 3：报告页面过长过乱

控制方式：

- 将跟训摘要、题页、答案页分层
- 打印时只取跟训相关内容，不把整份报告全带上

### 风险 4：家长使用路径不顺

控制方式：

- 功能留在 `本套报告` 内
- 提供直接打印入口
- 不要求额外跳页或再找别的菜单

## 11. 验收标准

如果这个功能做对了，应该能满足：

1. 家长批改完本套后，可以直接在 `本套报告` 里看到跟训页。
2. 跟训题不是原题重做，但明显针对同一个错误点。
3. 跟训题量短小，适合立即补练。
4. 打印时可以只打题页，也可以带答案一起打。
5. 题页和答案页都不破坏现有打印体验。
6. 孩子用完这页后，更容易把“刚错的知识点”补透，而不是只记住原题。

## 12. 推荐实现顺序

建议后续实施顺序如下：

1. 先把本套报告里的错题数据抽成“可用于变式生成”的输入
2. 再生成每位学生的本套变式训练块
3. 再接参考答案块
4. 最后接打印按钮与打印沙箱

这样做的好处是：

- 先保证训练逻辑成立
- 再保证家长可打印可使用
- 最后再打磨页面体验

## 13. 结论

本功能最合适的实现方式，不是新开系统，而是：

`在现有本套报告中，新增一个基于本套错题生成的变式跟训板块。`

它的价值不在于“再给一堆题”，而在于：

- 用本套刚刚暴露出的错误
- 立即生成同错误点的新题
- 让孩子马上补一轮

这是当前系统训练闭环里非常自然、也非常高价值的一步。
