# Mini Challenge 两阶段共同训练质量优化设计文档

> 状态：设计稿 v0.1
>
> 日期：2026-04-19
>
> 目标：在不破坏当前 `Advanced` 与 `Closure` 双 program 架构的前提下，为两个阶段建立一套共享的“高质量训练内核”，同步提升题目价值、错题回炉质量、变式辨别力和答案页复盘质量。

---

## 1. 这份文档要解决什么问题

当前系统已经具备两个训练阶段：

- 第一阶段：`Mini Challenge Advanced`
- 第二阶段：`Mini Challenge Closure`

并且已经具备这些重要能力：

- 稳定日训出卷
- 错题本回炉
- 同类变式
- 高价值 focus
- 两阶段独立画像
- 打印、批改、报告、同步闭环

但从“训练质量”角度看，系统仍然存在一个更高层的优化空间：

1. 很多题已经有覆盖，但“每天最值得练什么”还可以更精准；
2. 错题回炉已经存在，但还不够清楚地区分：
   - 原题会不会
   - 同类变式会不会
   - 放进综合里会不会
3. 两阶段现在都有训练逻辑，但缺少一套共享的质量语言；
4. 答案页虽然已经有步骤，但还可以更明确服务“下一次做对”；
5. 当前优化容易变成“局部补题”，缺少一条统一主线。

这份文档的目的，就是把这些零散优化收成一套共享的训练质量框架。

## 2. 总体判断

### 2.1 结论

下一轮最值钱的工作，不是继续单纯加题，而是建设一套共享的“训练质量内核”，让两个阶段在不同目标下共用同一种训练语言。

具体来说：

- `Advanced` 继续负责“算得准、算得稳、规则不过度漂移”
- `Closure` 继续负责“综合起来、迁移出去、完成小学计算总收束”
- 两个阶段共用：
  - 高价值误区家族
  - 三层回炉梯度
  - 讲解质量模板

### 2.2 为什么不是继续分散补题

如果继续沿用“发现哪里不够就补一类题”的方式，会有几个问题：

- 质量提升不成体系；
- 两阶段容易逐渐分叉；
- 新增题型难以判断它属于“基础稳定”还是“综合收束”；
- 文档、验证和后续维护都缺少中心结构。

所以这次应该先抽象共用训练质量层，再把它接回两个阶段。

## 3. 设计目标

本轮共同优化只追求 4 个结果：

1. `更高命中率`
   每天更容易练到最值钱、最容易反复出错的知识点。

2. `更高辨别力`
   能区分学生是“记住了原题”，还是“真正掌握并能迁移”。

3. `更高迁移性`
   训练从原题扩展到同类变式、反向题、边界题、综合题。

4. `更高复盘质量`
   答案页不只是告诉学生答案，而是帮助学生下一次更容易做对。

## 4. 设计原则

### 4.1 不破坏当前双阶段结构

保留：

- `Advanced`
- `Closure`
- 程序切换
- 两阶段独立画像
- 当前打印与批改链路

本轮不做的事：

- 不推翻现有 section 骨架
- 不重写 StorageDB
- 不改同步模型
- 不把两阶段重新揉成一个 program

### 4.2 共享质量语言，但不共享阶段职责

共享的是训练质量内核。

不共享的是阶段职责：

- `Advanced` 更偏“稳定化”
- `Closure` 更偏“整合与迁移”

### 4.3 先轻扩展，不做重型画像系统

这轮优先在现有题目对象和现有画像逻辑之上，做“轻量语义增强”。

不先做：

- 全新的题库存储层
- 新数据库
- 很重的 mastery engine

## 5. 共享训练质量内核

我建议这次新增 3 层共享能力。

### 5.1 高价值误区家族

先把跨阶段最值得长期追打的知识点，抽成统一题族。

第一批建议固定为 6 个：

1. `decimal_division`
   小数除法、小数点定位、估商、商小于 1、同倍扩大

2. `fraction_operation`
   分数意义、通分、约分、分数乘除、带分数处理

3. `conversion_bridge`
   分数 / 小数 / 百分数 / 比 的互化、比较与桥接

4. `unit_rate_speed`
   单位换算、量纲一致、速度、单价、平均量、率

5. `equation_method`
   未知数位置、逆运算顺序、比例方程、方法选择

6. `validation_estimation`
   估算、验算、数量级判断、结果合理性检查

这些 family 会作为共享层，对接现有 tag，而不是替代现有 tag。

### 5.2 三层回炉梯度

错题回炉从现在的“原题回炉 + 同类变式”继续升级成统一三层：

#### L1 原题回炉

作用：

- 检查学生是否记住了正确做法；
- 适合第一次错、刚建立修正时使用。

#### L2 同类变式

作用：

- 保持同知识点，但变数字、顺序、表达形式；
- 检查学生是否会迁移，而不是只记住原题。

#### L3 反向 / 边界 / 综合变式

作用：

- 把同一知识点放进更高辨别力的场景中；
- 测试学生是否真正理解。

例子：

- 小数除法：从“直接求商”变成“先判断商在 1 上下”
- 分数运算：从“直接算”变成“先比较再算”
- 互化题：从“转成分数”变成“哪个更接近 1”
- 单位率题：从“直接求值”变成“先统一单位再决定方法”

### 5.3 讲解质量模板

答案页统一升级成 3 种讲解视角：

#### 规则提醒

说明这一题最核心的规则是什么。

例如：

- 除数有小数，先把两数同乘 10 或 100；
- 百分数先转成分母 100 的分数，再约分；
- 带分数先化成假分数，再统一处理。

#### 方法提醒

说明这一题为什么该用这个方法，而不是只给计算过程。

例如：

- 这题不是硬算最快，而是先统一单位；
- 这题更适合先估商，再点小数点；
- 这题先找共同分母，比直接算更稳。

#### 检验提醒

说明做完以后该如何快速判断自己是不是大概率做对了。

例如：

- 结果应该比 1 大还是比 1 小；
- 单位是否统一；
- 数量级是否合理；
- 商或积是否和原数关系一致。

## 6. 两个阶段的职责分工

### 6.1 Advanced 的职责

`Advanced` 继续负责：

- 单点误区追打
- 计算稳定化
- 规则内化
- 让“会做”变成“稳定做对”

这一阶段更适合：

- 高频基础误区
- 易反复出错的规则点
- 小数除法、分数运算、互化比较的精准矫正

### 6.2 Closure 的职责

`Closure` 继续负责：

- 跨表示桥接
- 方法选择
- 综合迁移
- 站在六年级 / 初中视角完成小学计算总收束

这一阶段更适合：

- 单位 / 率 / 速度
- 跨表示综合
- 结果合理性检验
- 同一知识点的综合使用

### 6.3 共享内核如何分别接入

共享质量内核接到两个阶段，但职责不同：

- `Advanced` 使用它来提高“精准追打”的质量；
- `Closure` 使用它来提高“综合收束”的质量。

## 7. 第一批落地题族

这一轮不建议全量铺开，先做第一批高价值题族。

### 7.1 第一批建议范围

1. `小数除法`
2. `分数运算`
3. `互化与比较`

这三组优先的原因：

- 两个阶段都能用；
- 训练价值最高；
- 现有系统已经有比较成熟的 generator；
- 最容易看出“原题会”和“变式会”之间的差异。

### 7.2 第二批建议范围

等第一批稳定后，再继续覆盖：

4. `单位 / 率 / 速度`
5. `方程方法选择`
6. `估算 / 验算 / 结果检验`

## 8. 两阶段的接入方式

### 8.1 Advanced 接入方式

保留当前 daily paper 骨架，但新增两种轻量强化位。

#### 高价值回炉位

每套卷保留 1 个固定高价值回炉位：

- 优先从 6 个高价值 family 中挑选；
- 可以是 L1、L2 或 L3；
- 不再局限于 exact replay。

#### 检验型小题

每天增加 1 个轻量的检验型题：

- 数量级判断
- 商大小判断
- 单位合理性检查
- 结果接近值判断

目标不是变难，而是避免纯机械训练。

### 8.2 Closure 接入方式

保留当前六大区骨架，但升级现有重点强化逻辑。

#### 重点强化区升级

当前 `Section V` 已经有 focus 逻辑。

下一步升级为：

- 明确支持 L1 / L2 / L3
- 不只出 focus family 的原题或简单变式
- 也能出反向 / 边界 / 综合变式

#### 综合区嵌入反向题

在桥接区、率/单位区、混合区中，开始嵌入少量高辨别力题：

- 反向题
- 边界比较题
- 先判断再计算题

这样 `Closure` 就不会只是“综合卷”，而会更像真正的小学计算总收束。

## 9. 数据与实现边界

### 9.1 复用现有字段

优先复用：

- `tag`
- `reason`
- `errorBook`
- `step`
- `focusMeta`
- `history`
- `profile`

### 9.2 轻量新增字段

题目对象新增轻量语义：

- `qualityFamily`
- `replayLevel`
- `explanationMode`

可选辅助字段：

- `validationHint`

这些字段只为训练质量服务，不改变当前主存储结构。

### 9.3 不做的事

本轮明确不做：

- 新数据库
- 重型 mastery 图谱
- 全量题型重写
- 全系统 tag 重构

## 10. 实施顺序建议

建议固定按这个顺序做：

1. 先做共享 `qualityFamily`
2. 再做共享 `L1 / L2 / L3 replay`
3. 先接 `Advanced`
4. 再接 `Closure`
5. 最后统一升级答案页讲解模板

原因：

- 先统一训练语言；
- 再分别落进两个阶段；
- 最后再提升复盘质量；
- 这样返工最少。

## 11. 验收标准

这一轮完成后，应该满足以下结果：

### 11.1 训练质量

- 同一个高价值知识点可以稳定出现：
  - 原题回炉
  - 同类变式
  - 反向 / 边界 / 综合变式

### 11.2 Advanced 侧

- 第一阶段更像“精准追打”
- 不只是反复出原题
- 高频误区命中率更高

### 11.3 Closure 侧

- 第二阶段更像“综合收束”
- 不只是统一综合卷
- 更能检验迁移和方法选择

### 11.4 答案页

- 不只是答案和步骤
- 还要明确告诉学生：
  - 错在哪
  - 为什么这样做
  - 怎么快速检查

### 11.5 不破坏当前系统

必须保持稳定：

- 打印
- 错题本
- 云同步
- 两阶段切换
- 当前 set 缓存
- 当前批改与报告链路

## 12. 最小可交付版本

第一版建议只做到：

- 共享 `qualityFamily`
- 共享 `L1 / L2 / L3 replay`
- 先覆盖：
  - 小数除法
  - 分数运算
  - 互化与比较
- `Advanced` 先加高价值回炉位
- `Closure` 先升级重点强化区
- 答案页先给这三组题统一讲解模板

这样第一版就已经能明显提高训练质量，同时风险仍然可控。

## 13. 推荐结论

我推荐把这次优化正式定义为：

`Mini Challenge Shared Training Quality Core v1`

它不是一个新阶段，也不是一个新页面，而是一层同时服务 `Advanced` 和 `Closure` 的质量内核。

先把这层做对，后面无论继续强化第一阶段、深化第二阶段，还是扩展到更多小学计算题族，都会更稳、更清楚、更容易维护。
