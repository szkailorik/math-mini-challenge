# Mini Challenge 第二阶段前端体验设计文档

> 适用范围：
> `Mini Challenge Advanced` 与 `小学计算收束阶段` 共存时的页面与交互设计
>
> 目标：
> 在不破坏现有打印与训练闭环的前提下，让第二阶段拥有清晰、稳重、可理解的界面体验。

---

## 1. 设计目标

第二阶段的前端设计不是为了“做得更花”，而是为了把这 4 件事做清楚：

1. 让家长一眼知道当前在哪个训练阶段
2. 让第二阶段切换路径清楚，但不打断第一阶段使用习惯
3. 让第二阶段的训练目标、阶段状态、 readiness 更容易理解
4. 继续保证学生题卷保持考试卷式、打印优先、低干扰

## 2. 现有页面约束

当前页面已经形成稳定结构，必须尊重这些现实：

- 顶部控制区承担主要操作入口
- 中间 `paper-container` 承担题卷和答案页渲染
- 页面长期依赖：
  - `打印AB四页`
  - `打印答案`
  - `错题本`
  - `本套批改报告`
  - `知识点地图`
  - `数据备份 / Gist 同步`
- Question sheet 必须继续是打印主角
- 新增 UI 必须默认不进入打印

所以，第二阶段前端设计应该是：

- `增强控制区`
- `增强状态表达`
- `增强报告结构`
- `保持题卷克制`

## 3. 页面信息架构

建议第二阶段仍使用当前单页，不开新主页面。

推荐的信息架构从上到下分为 4 层：

### 第一层：全局控制层

位置：

- 页面顶部现有控制区

功能：

- set 切换
- 打印
- 错题本
- 报告
- 知识点地图
- 数据备份

要求：

- 这一层继续保持现有主操作习惯
- 第二阶段不重新发明导航

### 第二层：阶段状态层

位置：

- 控制区和题卷区之间

功能：

- 显示当前 program
- 显示阶段目标
- 显示 readiness / 当前建议动作

要求：

- 是“家长读”的区域
- 默认 `no-print`
- 信息要简洁，不抢主按钮

### 第三层：题卷层

位置：

- `paper-container` 主体

功能：

- 继续承载 question sheets 和 answer sheets

要求：

- 学生纸面保持干净、考试卷风格
- 不把 program 提示、阶段引导、ready 状态塞进题卷

### 第四层：复盘层

位置：

- 继续复用 modal / report 区

功能：

- 本套批改报告
- 错题本
- 知识点地图

要求：

- 第二阶段的价值更多体现在这里，而不是体现在题卷花样上

## 4. 核心新增组件

## 4.1 Program 切换条

### 作用

- 显示当前 program
- 提供 program 切换

### 布局建议

左侧：

- 小标签：`当前训练`
- 当前 program 名称

右侧：

- 轻量切换控件

建议文案：

- `Mini Challenge Advanced`
- `小学计算收束阶段`

### 视觉建议

- 高度低于主按钮区
- 采用细边框卡片，不做强烈背景块
- 当前 program 有清晰 accent，但不高饱和

### 交互建议

- 切换前若第二阶段未初始化，可弹出轻提示
- 第一阶段始终为默认态

## 4.2 阶段状态卡

### 作用

- 解释“当前处于哪个阶段”
- 告诉家长“现在该继续巩固，还是可以进入第二阶段”

### 信息结构

建议 4 行：

1. 当前阶段名
2. 当前阶段一句话目标
3. readiness 状态
4. 推荐动作

示例：

- `当前阶段：Mini Challenge Advanced`
- `目标：持续强化核心算法、互化、方程和策略训练`
- `阶段状态：已训练 26 天，接近第二阶段入口`
- `建议动作：继续巩固 3~5 套后可进入小学计算收束阶段`

### 视觉建议

- 用柔和底色卡片
- 不做复杂进度条第一版
- readiness 更适合用状态词而不是先用百分比

推荐状态词：

- `继续巩固`
- `接近切换`
- `可进入第二阶段`

## 4.3 第二阶段首次进入说明卡

### 作用

- 把“第二阶段不是更难，而是更完整、更稳”讲清楚

### 建议内容

- 标题：`欢迎进入小学计算收束阶段`
- 一句话说明：
  - `这一阶段会把整数、小数、分数、百分数、比、比例、单位与策略训练更系统地收束起来。`
- 保持原有习惯：
  - `每天仍是两页训练，系统仍会继续记录、批改和调训。`

### 交互建议

- 只在首次进入或版本升级后主动展开
- 后续可折叠成一行提示

### 视觉建议

- 比阶段状态卡稍醒目
- 但依然是 `no-print`

## 4.4 第二阶段报告摘要条

### 作用

- 让第二阶段报告从“错题罗列”升级成“训练质量摘要”

### 建议指标

- 算法稳定度
- 表示切换稳定度
- 方法选择稳定度
- 结果检查稳定度

### 呈现建议

第一版不要做复杂图表。

推荐：

- 4 个横向短指标卡
- 每个卡片包含：
  - 指标名
  - 状态词
  - 一句提醒

示例：

- `算法稳定度：稳定`
- `表示切换：需要加强`
- `方法选择：偶有误判`
- `结果检查：明显不足`

## 5. 视觉语言建议

## 5.1 风格关键词

第二阶段前端建议延续这些关键词：

- 稳
- 清楚
- 结构化
- 低干扰
- 打印友好

不建议的关键词：

- 华丽
- 儿童化装饰
- 高饱和升级感
- 仪表盘化过度

## 5.2 色彩建议

当前系统整体适合继续保持浅底、清晰边界、轻强调的方向。

第二阶段建议新增一组更“成熟”的 accent：

- 主 accent：深青 / 墨绿
- 辅助 accent：灰蓝
- 成功态：稳重绿色
- 注意态：琥珀色
- 风险态：砖红色

原则：

- accent 用于阶段与状态
- 不用于学生题卷主体

## 5.3 字体与层级建议

继续保持：

- 正文高可读
- 信息层级明确
- 数学内容优先

建议层级：

- L1：页面主标题 / program 名
- L2：状态卡标题 / 模块标题
- L3：正文说明
- L4：辅助状态字与注释

## 5.4 卡片和边框建议

建议控制区新组件使用：

- 中等圆角
- 轻边框
- 弱阴影或无阴影

不建议：

- 厚重投影
- 巨大渐变背景
- 强游戏化胶囊块

## 6. 题卷设计原则

第二阶段最重要的前端判断是：

`不要为了体现第二阶段，而把学生题卷做得更复杂。`

题卷仍应坚持：

- 考试卷式
- 低提示
- 强数学可读性
- 强打印稳定性

继续保留：

- 干净页眉
- 清晰题号
- 稳定分数与运算符样式
- 极少量必要题干提示

继续避免：

- 大量训练标签
- 大量状态提示
- 花哨 icon
- 高度不确定的说明块

## 7. 打印优先规范

第二阶段前端设计必须服从打印优先。

明确规定：

1. program 切换条不打印
2. 阶段状态卡不打印
3. 第二阶段首次说明卡不打印
4. 报告摘要条只出现在报告，不混入 question sheet
5. 第二阶段 question sheet 继续沿用现有 `.question-sheet` 结构

也就是说：

- 所有新增前端设计，优先存在于 `no-print` UI 层
- 学生纸面结构尽量不增加新的不确定高度

## 8. 响应式建议

虽然当前核心工作流偏桌面 + 打印，但第二阶段 UI 仍应兼顾手机和 iPad 查看。

### 桌面端

- Program 切换条与阶段状态卡可横向布局
- 控制区可两列或三列分组

### 平板端

- 控制区可分两行
- Program 切换条保持完整可读

### 手机端

- Program 切换条改为上下布局
- 阶段状态卡改为单列
- 文字优先，不追求同屏塞太多

## 9. 推荐实现顺序

如果按前端角度落地，我建议顺序是：

1. 先做 `Program 切换条`
2. 再做 `阶段状态卡`
3. 再做 `首次进入说明卡`
4. 最后做 `第二阶段报告摘要条`

原因：

- 前 3 个直接决定用户是否理解第二阶段
- 报告摘要条虽然重要，但属于第二阶段开始稳定运行后的增强项

## 10. 验收清单

前端设计是否达标，建议用这 8 条人工检查：

1. 家长是否能一眼看出当前在哪个阶段
2. 切换入口是否清楚但不抢主操作
3. 第二阶段是否看起来像现有系统的延伸，而不是新网站
4. 题卷是否依然干净像考试卷
5. 打印是否完全不受新增 UI 干扰
6. 第二阶段状态卡是否能看懂，不依赖专业术语
7. 第二阶段首次进入说明是否足够短，不烦人
8. 第二阶段报告是否更像“训练质量总结”，不只是错题表
