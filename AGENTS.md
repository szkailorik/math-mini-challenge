# Agents

## Project Goal

Build and maintain a grade 3-4 math fluency web app with adaptive practice, durable local-first progress, and light gamification.

## Roles

- 产品经理AG: requirements, scope, user stories, acceptance criteria.
- 学习科学AG: retrieval practice, spacing, interleaving, misconceptions, feedback loops.
- 设计AG: learning flow, grading UX, print layout, error-book experience.
- 架构AG: static app structure, storage model, sync model, deploy path.
- 文档AG: README, product spec, development spec, workflow notes.
- 开发AG: implement changes, keep edits scoped, confirm locally before asking.
- 测试AG: functional checks, edge cases, accessibility and print/export smoke tests.
- 审核AG / 质量审核AG: feasibility, correctness, learning alignment, release readiness.

## Handoff Format

Use these fields for each role handoff:

- What I changed
- Open questions
- Risks
- Next steps

## Iteration Rule

Run at least three internal passes for substantial work:

1. Thin slice end-to-end flow.
2. Quality, error handling, and adaptive rules.
3. UX polish, performance, docs, and edge cases.

## Research Alignment

The app must remain grounded in retrieval practice, spacing or interleaving, and effective feedback. Document major learning-design decisions in `PRODUCT_SPEC.md` and technical implications in `DEV_SPEC.md`.
